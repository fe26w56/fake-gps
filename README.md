# 位置偽装ツール (CLI版) へようこそ！

このツールは、スマートフォンの位置情報（GPS）を、あなたの好きな場所に変更するためのプログラムです。
「特定の店舗にチェックインするテスト」や、「ある場所に入ったときに通知が来るかどうかのテスト」などを、実際にその場所へ行かなくても行えるようにするために作られました。

このREADMEでは、ツールの使い方をステップバイステップで解説します。

---

## 1. このツールで何ができるの？

大きく分けて、以下の4つのことができます。

- **① 位置を固定する (`set`)**
  - スマートフォンの現在地を、指定した緯度・経度の場所に「瞬間移動」させます。
- **② 2点間を移動する (`move`)**
  - 「A地点」から「B地点」まで、指定した速さで歩いたり車で移動したりしているように見せかけます。
- **③ 元の場所を覚えておく (`save`)**
  - 位置偽装を始める前に、スマートフォンの「本当の」現在地を一時的に保存しておきます。
- **④ 覚えた場所に戻る (`restore`)**
  - 保存しておいた「本当の」現在地に、位置情報を戻します。

これらの機能は、`android_tool.py`（Android用）と`ios_tool.py`（iPhone/iPad用）という2つのプログラムを通じて利用できます。

## 2. 使う前に必要な準備

このツールを動かすには、お使いのパソコンとスマートフォンでいくつかの準備が必要です。

### パソコン側の準備（Windows, Mac共通）

1. **Pythonのインストール**
   - このツールはPythonというプログラミング言語で書かれています。もしPythonがパソコンに入っていなければ、[公式サイト](https://www.python.org/downloads/)から最新版（3.12以上）をインストールしてください。
2. **`haversine`ライブラリのインストール**
   - 2点間の距離を計算するために、特別なライブラリ（部品）が必要です。
   - コマンドプロンプト（Windows）やターミナル（Mac）を開き、以下のコマンドを打ち込んでEnterキーを押してください。
     ```bash
     pip install haversine
     ```

### スマートフォン側の準備

#### Androidの場合

1. **開発者向けオプションを有効にする**
   - `設定` > `デバイス情報` > `ビルド番号`を7回連続でタップします。
2. **USBデバッグを有効にする**
   - `設定` > `開発者向けオプション` > `USBデバッグ`をオンにします。
3. **ADBコマンドツールの準備**
   - [Android SDK Platform-Tools](https://developer.android.com/studio/releases/platform-tools)をダウンロードし、パソコンのどこかに解凍します。
   - そのフォルダのパスを、システムの環境変数`PATH`に追加してください。（この手順が難しい場合は、ウェブで「adb pathを通す」などで検索してみてください）

#### iPhone/iPad (iOS) の場合

1. **Xcodeのインストール**
   - MacのApp Storeから`Xcode`をインストールしてください。（容量が大きいので時間がかかります）
2. **Xcode Command Line Toolsのインストール**
   - ターミナルを開き、以下のコマンドを打ち込んでEnterキーを押してください。
     ```bash
     xcode-select --install
     ```
3. **開発者モードを有効にする**
   - iPhoneをMacにUSBで接続し、Xcodeを起動します。
   - `Window` > `Devices and Simulators`を開き、お使いのiPhoneを選択します。
   - "Enable Developer Mode"のようなボタンがあれば、それを押して有効化します。（Xcode 17以降では、物理デバイスの位置偽装もこのツールで行えるようになります）
   - **シミュレータを使う場合:** XcodeでiOSシミュレータを起動しておけば、特別な設定は不要です。

## 3. ツールの基本的な使い方

コマンドプロンプトやターミナルで、このツールが入っているフォルダに移動してから、以下のようにコマンドを打ち込みます。

### Androidでの使用例

まず、`python android_tool.py`に続けて、やりたいことに応じたコマンドを入力します。

- **東京駅に場所を固定する**
  ```bash
  python android_tool.py set 35.681236 139.767125
  ```
- **東京駅から秋葉原駅まで、歩く速さ（秒速1.5m）で移動する**
  ```bash
  python android_tool.py move 35.681236 139.767125 35.69836 139.773098 --speed 1.5
  ```
- **今の場所（例: 渋谷）を保存しておく**
  ```bash
  python android_tool.py save 35.65952 139.700475
  ```
- **保存した場所に戻る**
  ```bash
  python android_tool.py restore
  ```

### iOSでの使用例

iOSの場合は、どのシミュレータ（またはデバイス）を操作するかを指定するために`UDID`という識別子が必要です。

**UDIDの調べ方:**
Xcodeの`Window` > `Devices and Simulators`メニューで、対象のシミュレータやデバイスを選択すると、`Identifier`という項目に表示されています。

- **特定のシミュレータをロサンゼルスの位置に固定する**
  ```bash
  # <YOUR_SIMULATOR_UDID>の部分は、ご自身のUDIDに書き換えてください
  python ios_tool.py set <YOUR_SIMULATOR_UDID> 34.052235 -118.243683
  ```
- **シミュレータ上で移動する**
  ```bash
  python ios_tool.py move <YOUR_SIMULATOR_UDID> 34.052235 -118.243683 34.056235 -118.248683 --speed 3
  ```
- **位置の保存と復元（UDIDの指定が必要です）**
  ```bash
  python ios_tool.py save <YOUR_SIMULATOR_UDID> 34.052235 -118.243683
  python ios_tool.py restore <YOUR_SIMULATOR_UDID>
  ```

## 4. ログファイルについて

`move`コマンドで移動シミュレーションを行うと、その記録が自動的にCSVファイルとして保存されます。
`teleport_log_{実行日時}_{デバイスID}.csv`のようなファイル名で作成され、いつ、どのデバイスが、どの座標を通過したかの履歴を確認することができます。

何か問題があれば、このログファイルが原因調査の助けになります。
